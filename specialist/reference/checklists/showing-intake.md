# Showing-notes intake checklist

Use this when the agent says "capture showing notes" or pastes a dictation/transcript/summary from a property showing. Goal: produce a structured note that the agent can later cite during matching or pricing.

The note is **for the agent**, not for the buyer. Voice rule from `identity.md` applies — don't draft anything addressed to clients.

## Required from the agent (ask if missing)

- **Property** — address or listing identifier so the note can be linked back later
- **Date of showing**
- **Buyer** — short identifier (couple, family of 4, single relocator, repeat-tour buyer #2)
- **Who attended** — buyer alone, with partner, with kids, with third-party (parent, attorney, decorator)

## Pull from the dictation/transcript

### Buyer reactions

- **First impression at the door** — what they said or did stepping in
- **Specific features they reacted to** — both positive and negative; both are signal
- **Time spent in each room** — lingered = interest; rushed = friction or boredom
- **Questions they asked** — the question is data. What are they worried about?

### Friction points

- **Visible discomforts** — noise, light, smell, layout flow, neighbors, condition
- **Practical concerns raised** — commute, schools, parking, HOA, taxes, special assessments, walkability, climate
- **Comparisons made** — other properties they've toured + the comparison they drew

### Logistics
- **Timeline cues** — "we'd want to be in by August", "not in a rush"
- **Decision-makers** — anyone not present whose buy-in is needed
- **Money cues** — financing, contingencies, "would need to sell first"

## Ask the agent (their read, not the transcript)

- **Seriousness for THIS property:** 1–5 (1 = elimination, 5 = likely offer)
- **Next step:** second showing? written offer? eliminate? more research?
- **Homework before next conversation:** specific items (pull a comp, get permit history, ask listing agent about HOA reserves, etc.)

## Anti-patterns

- **Don't infer feelings the buyer didn't express.** "Buyer seemed disappointed by the kitchen" — only if the agent or transcript says so. Otherwise: "Buyer made no comment on the kitchen."
- **Don't blend transcript with agent read.** Mark each line as one or the other in the output. The transcript is what was *said*; the agent's read is what it *means*.
- **Don't write a paragraph.** Bullets, structured fields. The note is for retrieval, not narrative.

## Output

Produce the structured note per `templates/showing-note.md`. Surface "tags for retrieval" at the end so future matching/pricing jobs can find it.

## Reuse — why this matters

The showing note is not a one-shot artifact. It's input data:

- A future **matching job** for the same buyer can cite: "Per the [date] showing at [address], buyer flagged [X] as friction — Listing C may have the same issue."
- A future **pricing job** in the same micro-market can cite: "Multiple showing notes flag [Y] as friction in this corridor; consider conservative pricing on properties exhibiting it."

Treat each showing note as a small data point that compounds.
